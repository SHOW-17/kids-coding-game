# 🦄 あそびのもり

小学校1年生（6〜7歳）が、**文字を読まずに**遊びながら賢くなれる、こども向けのミニゲーム集（アーケード）。

オープニング画面（`index.html`）でゲームを選び、それぞれの遊びに入る。すべて **指タップ前提・ひらがな＋アイコンのみ・ビルド不要**（バニラJSの単一HTML）。

> **デザインテーマ「ねんどの森」** — ねんど（クレイ）／フェルトの質感の世界。マスコットはユニコーン（ブルー＝主役／グリーン＝はかせ／パープル＝ごほうび）。アイコンは絵文字でなく CSS図形、キャラは `assets/uni/` の画像で表現する。視覚言語の定義は [Design.md](Design.md) を参照（新画面はこれに従う）。

- **対象**：小1（漢字を読めない前提）。UIはひらがな＋アイコンのみ。
- **ねらい**：知的好奇心をくすぐり、非認知能力（集中・やりぬく力・試行錯誤・空間認識・創造性）を伸ばす。
- **やさしさ**：失敗しても叱らず、何度でも試せる。クリアは効果音・アニメ・紙吹雪で「できた！」を強く演出。
- **じゆうに あそべる**：どのゲームも好きなレベル／むずかしさを選んで遊べる（後述）。
- **品質**：プロ品質を目標に、見た目・演出・手触りまで作り込む。

## あそべるゲーム

| ゲーム | 仕組み | 育てる力 |
|---|---|---|
| **プログラミング** | やじるしを並べて▶。命令どおりにユニコーンが動く | シーケンス・デバッグ・論理 |
| **まねっこ**（サイモン） | 光った順番を覚えてまねする | 記憶・集中・やりぬく力 |
| **きまり あそび** | 並びの「つぎは なに？」を当てる | 規則性の発見・推理 |
| **かたち づくり** | ピースを影にはめて形を作る | 空間認識・創造性・試行錯誤 |
| **ぴたごら** | 坂を置いて玉をゴールへ転がす | 因果・計画・試行錯誤 |

「プログラミング」のレベル構成や設計詳細は [CLAUDE.md](CLAUDE.md) を参照。

### レベル／むずかしさを じゆうに えらべる

- **レベルのあるゲーム**（プログラミング・かたちづくり・ぴたごら）：ステータスバーの **◀ ▶** で前後のレベルへ自由に移動できる（ロックなし＝1〜最終レベルまでどこへでも）。進捗の **★／●ドットを直接タップ**しても飛べる。選んだレベルは保存され、次に開いたときも続きから。
- **レベルのないゲーム**（まねっこ・きまり）：個別レベルが無く難度が自動で上がる仕組みのため、代わりに **「むずかしさ えらび」** を用意。まねっこ＝光る速さ（ゆっくり／ふつう／はやい）、きまり＝出題の難しさ（やさしい／ふつう／むずかしい）。選択は保存される。

### 最高記録はホームで確認

アーケード（`index.html`）の各ゲームカードに、その子の **最高記録** を表示する。

| ゲーム | ホームに出る記録 |
|---|---|
| プログラミング | クリア数バー（◯/19）＋「さいこう レベルN」（到達した最高レベル） |
| まねっこ | 「さいこう Nだん」 |
| きまり | 「れんぞく Nもん」（連続正解の最高） |
| かたちづくり | 「Nこ かんせい」 |
| ぴたごら | 「Nこ クリア」 |

## 起動方法

ビルド不要。次のいずれかで `index.html` を開く。

```bash
# そのままブラウザで開く
open index.html            # mac
xdg-open index.html        # linux

# 簡易サーバーで配信（スマホ実機・Tailscale 等から見る場合）
python3 -m http.server 9090 --bind 0.0.0.0
# → http://<サーバーのIP>:9090/
```

`index.html` がアーケード（オープニング）。各ゲームは `games/*.html`。

## 技術メモ

- バニラJS／ビルド不要。共通基盤（`assets/*`）の上に各ゲームを単一HTMLで実装。
- フォント：Google Fonts（Mochiy Pop One / M PLUS Rounded 1c）。
- 効果音：Web Audio で生成（音声ファイル不要、try/catch でガード）。
- BGM：画面ごとに割り当ててループ再生（`assets/bgm.js`）。右上の音符ボタンで ON/OFF（全画面共通で保存）。
  `assets/bgm/<画面名>.mp3` を置けばその曲を優先再生、無ければやさしい合成ループを暫定で鳴らす。
  音源 mp3 は `.gitignore` 済み（公開リポジトリに市販曲を入れない）。詳細は [`assets/bgm/README.md`](assets/bgm/README.md)。
- 永続化：`localStorage`（ゲーム別の名前空間。使えない環境ではメモリに自動フォールバック）。
  メニュー左上の「せってい」（歯車）→「きろくを ぜんぶ けす」で全消去（子どもが誤操作しないよう設定内に格納）。
- アクセシビリティ：`prefers-reduced-motion` を尊重。色だけに頼らない表現。
- 動作確認：`google-chrome-stable` + puppeteer-core で各ページの console error / 例外 / 横スクロールはみ出しを検査（`scripts/test-games.js`。実行手順はファイル冒頭のコメント参照）。

### デザイン素材について
メニュー画面（`index.html`）は、ロゴ・背景・ユニコーン・ゲーム看板・カードアート・ボタンを
**画像素材（`assets/menu/*.webp`）** で構成し、チープ感を排した作り込みにしている。
各ゲーム画面（`games/*.html`）のアイコン類は引き続き CSS/SVG で「ねんど化」（**絵文字は使わない**）。
画像素材は LLM 画像生成（ChatGPT / Claude Design 等）で用意し、白背景は切り抜き → WebP 最適化して配置する運用。
マスコットのユニコーンは [Design.md](Design.md) の配役ルール（ブルー＝主役／グリーン＝はかせ／パープル＝ごほうび）に従う。
