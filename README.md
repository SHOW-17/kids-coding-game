# 🐻 くまさんプログラミング

小学校1年生（6〜7歳）が、**文字を読まずに**「順番に命令する」というプログラミングの核を学べるブラウザゲーム。

矢印などの命令を並べて ▶ を押すと、キャラクターが命令どおりに動く。ゴール（🍯）に着けばクリア。

- **対象**：小1（漢字を読めない前提）。UIはひらがな＋アイコンのみ。
- **操作**：指タップ前提の大きなボタン。スマホ・タブレットで快適。
- **やさしさ**：ぶつかっても叱らず自動でやり直し。何度でも試せる＝デバッグ練習。
- **ビルド不要**：バニラJSの単一HTML。ブラウザで開くだけで動く。

> キャラクターは仮（クマ🐻）。ゲームロジックはキャラに依存させない方針（差し替え予定）。

---

## あそびかた

1. 下の矢印ボタン（⬆⬇⬅➡）を押して、くまさんへの「めいれい」を並べる。
2. ▶ スタート！ を押すと、命令どおりにくまさんが動く。
3. ゴール（🍯）に着けばクリア。紙吹雪＆効果音で「できた！」を演出。
4. ぶつかったら自動でスタートに戻る。命令はそのまま残るので、直してもう一度。

レベルが進むと、新しい命令や仕掛けが少しずつ解禁される。

---

## 起動方法

ビルド不要。次のいずれかで開く。

```bash
# そのままブラウザで開く
open index.html            # mac
xdg-open index.html        # linux

# 簡易サーバーで配信（スマホ実機・Tailscale 等から見る場合）
python3 -m http.server 9090 --bind 0.0.0.0
# → http://<サーバーのIP>:9090/
```

`index.html` は実体 `kuma-programming.html` へのシンボリックリンク。

## 学べる概念（段階的に登場）

| # | 概念 | 仕組み |
|---|---|---|
| 1 | シーケンス（順番） | 命令を並べて実行する＝核 |
| 2 | デバッグ | 失敗→直す→再実行 |
| 3 | ループ 🔁 | 同じ動きをまとめる |
| 4 | コマンドの種類 🦘 | ジャンプ＝動きには種類がある |
| 5 | 計画・段取り 🍓 | いちごを集めてからゴール |
| 6 | 状態・因果 🔘🚪 | ボタンを押すととびらが開く |
| 7 | 空間把握 🌀 | ワープで対の出口へ |
| 8 | 因果・取り返し 📦 | はこを🎯へ。詰んだらやり直す |

5〜8 は**新しいボタンを増やさず**、盤面の仕掛けとして導入している（既存の移動命令で遊べる）。

## レベル（全19）

| Lv | 概要 |
|---|---|
| 1〜4 | 移動の基礎（順番・迷路） |
| 5〜6 | 🔁ループ（ブロック数制限で「まとめる理由」を作る） |
| 7〜10 | 🦘ジャンプ＋総合 |
| 11〜12 | 🍓いちご収集（計画） |
| 13〜14 | 🔘ボタン＆🚪とびら（因果） |
| 15〜16 | 🌀ワープ（空間） |
| 17〜19 | 📦はこおし（倉庫番ライト） |

レベルは `LEVELS` 配列で定義。追加・調整しやすい。
**レベルを足すときは必ず BFS で解の存在を確認する**（仕掛けレベルは状態にいちご回収・とびら開閉・箱位置を含める）。

## 技術メモ

- 単一HTML（`kuma-programming.html`）／バニラJS／ビルド不要。
- フォント：Google Fonts（Mochiy Pop One / M PLUS Rounded 1c）。
- 効果音：Web Audio で生成（try/catch でガード）。
- 永続化：`localStorage`（クリア状況・現在レベル・案内表示済みフラグ）。「🔄さいしょ」で初期化。
- キャラ・箱は絶対配置トークン。cell サイズは画面幅から動的計算（スマホ対応）。
- ファビコン／ホーム画面アイコン：クマ🐻＋はちみつ🍯の `icon.svg` から各サイズの PNG・ICO を生成。PWA マニフェスト（`manifest.webmanifest`）でスマホの「ホーム画面に追加」に対応（standalone 表示・名前「くまプロ」）。テーマ差し替え時は `icon.svg` を直して再変換する。

詳しい設計方針・制約は [CLAUDE.md](CLAUDE.md) を参照。
