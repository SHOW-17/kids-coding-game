# あそびのもり — デザインシステム「ねんどの森」

小学校1年生（6〜7歳）が、**文字を読まずに**遊びながら賢くなれる、こども向けミニゲーム集。
このドキュメントは、トップ画面と全ゲームを貫く視覚言語の定義です。新しい画面やゲームを足すときは、ここに従えば世界観が崩れません。

---

## 1. コンセプト

> **ねんど（クレイ）／フェルトの質感でできた、ちいさな森。**
> ぷっくり立体的で、手でさわれそうなあたたかさ。やさしくて、わくわくして、かしこくなれる。

- **キーワード**：わくわく・たのしい／かしこくなれそう・知育感／あんしん・あたたかい
- **絵本のような一場面**を各画面でつくる。机に向かう、丘で手をふる、星をならべる——「シーン」を見せる。

---

## 2. マスコット（ユニコーン）

森の住人。画像アセットは `assets/uni/`（実機）/ `assets/uni/`（このプロジェクト）。
**絵文字は使わない** — キャラは必ずこのユニコーン画像で表現する。

| 役 | 色 | 性格 | おもな出番 |
|---|---|---|---|
| **主役** | ブルー | 元気・好奇心 | トップのヒーロー、プログラミングの主人公、案内役 |
| **はかせ** | グリーン（メガネ） | かしこい・教える | レベルアップ案内、ヒント、推理系ゲーム |
| **げんき** | パープル | あかるい・ごほうび | クリア祝い、まねっこの中央、ゴールのともだち |

### 配役のルール
- **ゴール／ともだち**＝パープルの顔（`purple_face.png`）か手をふる姿（`purple_wave.png`）
- **クリア・ぜんぶ達成**＝パープルが手をふる（`purple_wave.png`）
- **新要素の説明・ヒント**＝グリーン（はかせ）
- **失敗・おしい**＝ブルーが考える（`blue_thinking.png`）— 責めない、励ます
- ヒーローでは3体を並べて「森のなかま」感を出す（主役を中央・大きく、脇役を小さく左右に）

### よく使うポーズ
`*_wave`（手をふる）/ `*_standing`（立ち）/ `*_sit`（すわり）/ `*_thinking`（考える）/ `*_face`（顔アップ）/ `blue_workdesk`（机に向かう・背景込み）

## 4. タイポグラフィ

| 役 | フォント | 使いどころ |
|---|---|---|
| 見出し・ロゴ・ボタン | **Mochiy Pop One** | ロゴ、ゲーム名、「あそぶ」「スタート！」、数字ピル |
| 本文・説明・ステータス | **M PLUS Rounded 1c**（700/800） | せつめい文、ヒント、ラベル |

- 文字はすべて**ひらがな・カタカナ**（漢字なし）。「よめなくても あそべる」ことが最優先。
- 本文は最低でも 12.5px、ヒント/見出しは 15〜30px。タップ説明より**色・形・動き**で伝える。
- ロゴは多重 text-shadow で**ねんどの立体押し出し**を表現（下方向に色を積層→影）。

## 5. かたち・質感・余白

### ねんどカードの作り方（最重要レシピ）
立体感は「**上にハイライト＋下に内側の影＋外に落ち影**」の3点セット。
```css
border-radius: 30px;            /* 大きく丸く。角は常にやわらかい */
box-shadow:
  0 16px 26px -12px rgba(150,100,40,.34),  /* 落ち影（地面）   */
  0 3px 0 rgba(150,100,40,.10),            /* 厚みのふち       */
  inset 0 3px 0 rgba(255,255,255,.9),      /* 上のハイライト   */
  inset 0 -10px 16px rgba(190,150,90,.12); /* 下の内側の影     */
```
- **押す**と `translateY(4px) scale(.985)` で“へこむ”。ボタンは下に厚み影 `0 5px 0` を必ず付ける。
- 背景に薄いフェルトの粒子ノイズ（SVG turbulence, `mix-blend-mode:multiply`, opacity .4〜.5）。
- 角丸 `--r:30px` / `--r-sm:20px`。とがった角は使わない。

### アイコンは絵文字でなくCSSで
星・丸・三角・ひし形・六角・矢印・うずまき・的・はこ・とびら・ボタン…すべて
`clip-path` / `border` / `conic-gradient` / `mask` で作る。各図形にも
`inset` ハイライト＋下影を入れて「ねんど化」する。

### モーション
- イージング：`--ease:cubic-bezier(.22,1,.36,1)` / 跳ね `--spring:cubic-bezier(.34,1.56,.64,1)`
- キャラは ふわふわ上下（bob, 3s前後）。登場は `.enter`（下から rise、段差 d1〜d6）。
- 正解＝キラッ（sparkles/ring）＋効果音。失敗＝やさしく shake、責めない文言。
- `prefers-reduced-motion` を必ず尊重（装飾アニメは止め、最終状態を見せる）。

## 6. レイアウトと画面の型

### 共通構造
1. **トップバー**：おうち（🏠ボタン＝CSS mask）＋ キャラ顔＋タイトル
2. **ステータス**：レベル／スコア／さいこう記録の丸ピル（Mochiy Pop One）
3. **ヒント1行**：いまやることを短く
4. **あそびの舞台**：盤面・ダイヤルなど。ねんどのトレイに乗せる
5. **操作部**：大きな丸ボタン。最小ヒット 44px、実質は 56〜84px
6. **結果ダイアログ**：キャラが上から飛び出す丸いカード。「おうち」＋「つぎへ」

### トップ画面のカード階層
- **フィーチャー**（プログラミング）：横長・王冠・進捗バー・「あそぶ」ボタン・「いちばん にんき」リボン
- **グリッド**（2列）：各ゲームのミニ看板（CSS図形＋キャラ）＋ のうりょくタグ ＋「しんさく！」リボン

---

## 7. コピーライティング

- 全文ひらがな・カタカナ。短く、やさしく、命令形はやわらかく（「〜してね」「〜しよう」）。
- 失敗は**励まし**：「おしい！」「ドンマイ！」「もういちど みてみよう」。叱らない。
- 成功は**素直なよろこび**：「やったね！」「せいかい！」「すごい！」「ばっちり！」
- のうりょくは前向きに：きおく力／すいり力／かたち力／くふう力。

---

## 8. アクセシビリティ・配慮

- **色だけに頼らない**：まねっこは色＋形、きまりは色トークンと形トークンを併用。
- ヒット領域は 44px 以上。文字より**アイコン・形・色・動き**で意味を伝える。
- すべての操作に**音**と**手応え（へこみ・振動）**のフィードバック。
- `prefers-reduced-motion` 対応、画像 `alt`/装飾は空 alt、見出し階層を保つ。

## 9. ファイル構成

```
index.html                 ← トップ（あそびのもり）
kuma-programming.html      ← プログラミング（自己完結／assets JS不要）
games/
  manekko.html             ← まねっこ
  kimari.html              ← きまり
  katachi.html             ← かたち
  pitagora.html            ← ぴたごら
assets/
  uni/                     ← ユニコーン画像（blue_/green_/purple_*）
  audio.js fx.js save.js shell.js  ← 共通基盤（実機は本物、プレビューは軽量フォールバック）
```

### 共通基盤の契約（グローバル）
- `Sfx`：`tap/pop/step/ding/note/success/fanfare/error/whoosh/sparkle`
- `FX`：`ring/burst/sparkles/confetti(celebrate)/shake`
- `Save.game(ns)`：`get/set/bestMax/bestMin`、`Save.resetAll([ns…])`
- `App`：`go(href)/toast/el`（任意で `showModal`）

> ゲーム側は `try/catch` でガードして呼ぶ（基盤が無くても落ちない）。
